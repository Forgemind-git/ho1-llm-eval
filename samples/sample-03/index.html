Data Extraction Benchmark

Test how accurately each model extracts structured JSON fields from unstructured customer text.

Problem Statement

Extract JSON with these fields: customer_name, order_id, issue, urgency_level (low/medium/high). Text: "Hi, I am Maria. Order #ORD-2847 arrived yesterday but three items are missing. I have a client event tomorrow morning."

What You're Measuring

CriterionWhat to look forScale
All 4 fields presentDoes the output include customer_name, order_id, issue, and urgency_level?yes/no
urgency_level correctly inferredShould be "high" — client event tomorrow makes it urgent.yes/no
Valid JSON outputCan the output be parsed by JSON.parse() without errors?yes/no
Speed feelSubjective: did the response feel fast or slow?fast/slow

Run the Test

No API key needed. Run this test manually in two tools and compare results.
  1. LM Studio Open LM Studio, load any model, go to the Chat tab.
  2. LM Studio Paste the prompt above. Check whether all 4 JSON fields appear, whether urgency is "high", and whether the JSON is valid. Note the response time feel.
  3. Claude.ai Open claude.ai in a new tab, start a new conversation.
  4. Claude.ai Paste the same prompt. Compare the JSON quality and urgency inference.
  5. Score Rate each model using the scorecard below.
